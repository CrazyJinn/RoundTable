# 需求确认报告

> 本文档用于确认对游戏需求的理解，列出潜在问题和后续开发建议。

---

## 一、需求理解

### 1.1 游戏概述

**游戏名称：** 废土之战（暂定）

**游戏类型：** 2D俯视角动作游戏（Top-down Action Game）

**引擎选择：** Cocos Creator

**核心主题：** 废土世界，科技与魔法的对立与共存

### 1.2 世界观设定

**历史背景：**
- 核战后文明崩塌，世界支离破碎
- 幸存人类分化为两大阵营

**力量分化：**
- 科技派：依靠科技武装，外骨骼机甲、枪械武器
- 魔法派：因变异觉醒魔法天赋，寿命延长至300岁

**冲突根源：**
- 资源匮乏，两大阵营为了生存不断争夺

### 1.3 核心玩法

**玩法循环：**
```
探索大地图 → 遭遇怪物 → 战斗获胜 → 搜集物资
      ↑                                    ↓
      └────────── 返回城镇强化/购买 ←───────┘
```

**主要活动：**
1. **探索：** 在三大地图区域自由探索
2. **战斗：** 与变异怪物战斗，使用技能和武器
3. **搜集：** 获取战斗掉落的物资
4. **强化：** 在城镇中使用物资强化技能
5. **任务：** 完成各组织发布的任务

### 1.4 双主角设定

| 主角 | 性别 | 年龄 | 隶属 | 背景 |
|------|------|------|------|------|
| 科技主角 | 男 | 17岁 | 骑士团 | 3岁时双亲被女主误杀，只有零碎记忆 |
| 魔法主角 | 女 | 120岁（外貌20岁） | 秘术协会 | 14年前任务中误杀男主双亲 |

**身世冲突：** 两名主角有深层的剧情冲突（杀亲之仇），这是游戏的核心剧情线索。

### 1.5 势力组织

**科技派（3个）：**
- 金属合唱团（深红，枪械远程）
- 骑士团（黑白灰，外骨骼近战）
- 机械风暴（**待定**，召唤机器人）

**魔法派（3个）：**
- 教廷（金色白色，祷告近战）
- 秘术协会（彩虹色，魔法远程）
- 灵魂行者（银色黑色，召唤兽）

### 1.6 操作方式

| 按键 | 功能 |
|------|------|
| WASD | 移动（8方向） |
| 空格 | 翻滚（无敌帧） |
| 左键 | 普通攻击/射击 |
| 右键 | 终极技能 |
| R | 换子弹/冥想 |
| Q | 技能1 |
| E | 技能2 |
| F | 拾取 |

**鼠标：** 焦点锁定目标，左右键操作

## 三、潜在问题

### 3.1 已识别问题

| 问题ID | 问题描述 | 影响范围 | 优先级 | 建议解决方案 |
|--------|----------|----------|--------|--------------|
| ISSUE_001 | 目标平台未确认，影响分辨率和性能优化 | 全局 | 高 | 确认目标平台（PC/移动端/网页端） |
| ISSUE_002 | 机械风暴组织设计主题待定 | 场景、UI | 中 | 确定该组织的设计风格（建议：金属蓝/铜色） |
| ISSUE_003 | 毒液蜘蛛外观描述缺失 | 角色美术 | 中 | 补充详细外观描述（建议：黑色、紫色毒液、多眼） |
| ISSUE_004 | 双主角剧情线结构未定义 | 剧情设计 | 高 | 确认剧情结构（单/双/可选） |
| ISSUE_005 | 关卡数量和流程未定义 | 关卡设计 | 高 | 规划关卡结构和数量（建议：10-15关） |
| ISSUE_006 | 技能系统具体内容未定义 | 游戏设计 | 高 | 设计各技能效果和升级路线 |
| ISSUE_007 | 物资系统细节未定义 | 游戏设计 | 高 | 规划物资类型、获取方式、用途 |
| ISSUE_008 | NPC和任务内容未定义 | 剧情设计 | 中 | 设计NPC角色和任务线 |
| ISSUE_009 | 游戏时长目标未定义 | 项目规划 | 中 | 确定游戏时长（建议：2-4小时） |
| ISSUE_010 | 结局设计未明确 | 剧情设计 | 高 | 设计单结局/多结局/分支剧情 |

### 3.2 技术风险

| 风险ID | 风险描述 | 影响范围 | 缓解措施 |
|--------|----------|----------|----------|
| RISK_001 | 大量同屏怪物可能导致性能问题 | 战斗场景 | 限制同屏怪物数量、优化精灵图 |
| RISK_002 | 8方向动画可能增加美术工作量 | 角色动画 | 可先做4方向，后期升级 |
| RISK_003 | 场景切换可能导致加载延迟 | 全局 | 预加载资源、添加加载界面 |
| RISK_004 | 手绘风格美术资源可能较难获取 | 美术 | 准备参考素材、使用AI辅助生成 |

### 3.3 设计风险

| 风险ID | 风险描述 | 影响范围 | 缓解措施 |
|--------|----------|----------|----------|
| RISK_005 | 双主角选择可能导致剧情割裂 | 剧情设计 | 考虑共通剧情线+分支剧情 |
| RISK_006 | 战斗系统深度可能不足 | 游戏性 | 设计多样的怪物AI和技能组合 |
| RISK_007 | 探索内容可能单调 | 游戏性 | 添加隐藏要素、随机事件 |
| RISK_008 | 物资系统可能失去平衡 | 游戏性 | 进行数值平衡测试和调整 |

## 四、后续开发建议

### 4.1 第一阶段：核心原型（2-3周）

**目标：** 创建可玩的核心玩法原型

**任务清单：**
- [ ] 确认目标平台和分辨率
- [ ] 创建科技主角完整精灵图和基础动画
- [ ] 创建1-2种基础怪物（辐射巨鼠、钢鬃野猪）
- [ ] 创建1个战斗场景背景（废土）
- [ ] 实现基础移动系统（WASD + 鼠标瞄准）
- [ ] 实现基础攻击系统（左键普通攻击）
- [ ] 实现基础怪物AI（追踪、攻击）
- [ ] 实现基础UI（血条、技能栏）
- [ ] 添加基础音效（攻击、受伤）

**验收标准：**
- 可以控制主角移动
- 可以攻击怪物
- 怪物可以追踪和攻击主角
- 主角死亡可以重生
- 基本流畅运行

**目标：** 完善游戏内容

**任务清单：**

- [ ] 实现城镇场景和交互功能
- [ ] 实现完整的UI系统
- [ ] 实现技能系统（Q、E、右键）
- [ ] 实现翻滚系统（空格）

**目标：** 打磨游戏体验

**任务清单：**
