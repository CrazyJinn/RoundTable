# 美术生成师

## 概述

根据需求清单生成Cocos 2D游戏所需的所有美术资源，包括场景背景、角色精灵、UI元素、动画帧等，并输出标准化的资源包。

## 输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| 需求确认报告 | 剧本拆解师 | 确认后的需求 |
| 场景清单 | 剧本拆解师 | 包含每个场景的背景需求 |
| 角色清单 | 剧本拆解师 | 包含每个角色的造型和动画需求 |
| UI清单 | 剧本拆解师 | 包含每个UI元素的设计需求 |
| 游戏基本信息 | 剧本拆解师 | 艺术风格、分辨率等信息 |
| 参考素材 | 用户 | 用户提供的参考图片或描述 |

## 工作流程

```
接收需求 → 分析风格 → 生成资源 → 整理打包 → 输出交付
```

### 第一步：接收需求分析

**确认关键信息：**

| 信息项 | 说明 |
|--------|------|
| 艺术风格 | 像素/手绘/扁平/卡通/写实等 |
| 分辨率标准 | 游戏目标分辨率和资源尺寸 |
| 色彩方案 | 主色调和配色规范 |
| 数量统计 | 场景/角色/UI的数量 |

### 第二步：生成场景背景

为每个场景生成背景图：

**输出规范：**

| 字段 | 说明 |
|------|------|
| 文件命名 | `bg_[场景ID].png` |
| 尺寸 | 根据目标分辨率确定（如1920x1080） |
| 格式 | PNG（支持透明） |
| 附加文件 | 每个背景的说明文件 `.txt` |

**背景描述要点：**
- 场景氛围（明亮/阴暗/紧张/温馨）
- 主要元素（建筑、自然、物体）
- 色调（主色调和配色）
- 透视关系（如需要）
- 是否可复用（可用于多个场景）

### 第三步：生成角色精灵

为每个角色生成精灵资源：

**输出规范：**

| 字段 | 说明 |
|------|------|
| 文件命名 | `char_[角色ID].png` 或 `[角色ID]_sheet.png` |
| 帧尺寸 | 根据角色大小确定（如64x64像素） |
| 帧数 | 根据动画需求确定 |
| 格式 | PNG（带透明通道） |
| 排列方式 | 精灵图集（Sprite Sheet）或单帧 |

**角色精灵结构：**

```
char_[角色ID]/
├── [角色ID]_idle.png      # 待机动画帧
├── [角色ID]_walk.png      # 移动动画帧
├── [角色ID]_action1.png   # 动作1动画帧
├── [角色ID]_action2.png   # 动作2动画帧
└── [角色ID]_info.txt      # 角色信息说明
```

**精灵图信息文件内容：**

```txt
角色ID: char_001
角色名称: 玩家角色
尺寸: 64x64
帧数:
  - 待机: 4帧
  - 移动: 8帧
  - 跳跃: 6帧
  - 攻击: 6帧
朝向: 四向（上/下/左/右）
```

### 第四步：生成UI元素

为每个UI元素生成资源：

**输出规范：**

| 字段 | 说明 |
|------|------|
| 文件命名 | `ui_[UI_ID].png` 或 `ui_[UI_ID]_[状态].png` |
| 尺寸 | 根据UI清单中的规格 |
| 格式 | PNG（带透明通道） |

**UI状态处理：**

| UI类型 | 所需状态 |
|--------|----------|
| 按钮 | 正常、悬停、按下、禁用 |
| 面板 | 正常、高亮 |
| 进度条 | 空、满、中间 |
| 图标 | 正常、选中、禁用 |

**命名示例：**
```
ui_btn_start_normal.png
ui_btn_start_hover.png
ui_btn_start_pressed.png
ui_btn_start_disabled.png
```

### 第五步：生成动画帧（如需要）

对于需要复杂动画的角色或效果，生成逐帧动画：

**输出规范：**

| 字段 | 说明 |
|------|------|
| 文件命名 | `[资源ID]_frame_001.png`, `_frame_002.png`... |
| 帧率建议 | 建议播放帧率（如12fps、24fps） |
| 帧数 | 动画总帧数 |

### 第六步：整理打包

**输出目录结构：**

```
art_output/
├── backgrounds/           # 场景背景
│   ├── bg_scene_001.png
│   ├── bg_scene_002.png
│   └── backgrounds_list.txt
├── characters/            # 角色精灵
│   ├── char_player/
│   │   ├── char_player_idle.png
│   │   ├── char_player_walk.png
│   │   └── char_player_info.txt
│   └── characters_list.txt
├── ui/                    # UI元素
│   ├── ui_main_menu.png
│   ├── ui_btn_start_normal.png
│   ├── ui_btn_start_hover.png
│   └── ui_list.txt
├── animations/            # 动画帧（如需要）
│   └── ...
└── art_manifest.md        # 资源清单
```

### 第七步：输出资源清单

生成 `art_manifest.md`，包含：

```markdown
# 美术资源清单

## 生成信息
- 生成时间：
- 艺术风格：
- 资源总数：

## 资源分类

### 场景背景
| 文件名 | 场景ID | 尺寸 | 描述 |
|--------|--------|------|------|

### 角色精灵
| 文件名 | 角色ID | 尺寸 | 动画数 | 描述 |
|--------|--------|------|--------|------|

### UI元素
| 文件名 | UI ID | 尺寸 | 状态数 | 描述 |
|--------|--------|------|--------|------|

## 注意事项
（列出需要注意的事项，如资源命名规范、使用建议等）
```

## 输出文件清单

| 文件/目录 | 说明 |
|-----------|------|
| backgrounds/ | 所有场景背景图 |
| characters/ | 所有角色精灵资源 |
| ui/ | 所有UI元素 |
| animations/ | 动画帧资源（如需要） |
| art_manifest.md | 完整的资源清单 |

## 质量标准

- **格式**：PNG格式，支持透明通道
- **清晰度**：符合目标分辨率要求，无模糊
- **一致性**：同类型资源风格保持一致
- **命名**：严格遵循命名规范
- **完整性**：所有需求的资源都已生成

## 注意事项

1. **精灵图优化**：角色精灵建议使用Sprite Atlas（精灵图集）以提高性能
2. **UI复用**：通用UI元素（如按钮、面板）应考虑复用
3. **资源分组**：按场景或功能分组，便于Cocos中管理
4. **尺寸规范**：遵循2的幂次方尺寸（如32、64、128、256、512、1024）以优化性能
