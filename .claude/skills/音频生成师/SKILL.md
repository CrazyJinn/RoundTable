# 音频生成师

## 概述

根据音频需求清单生成Cocos 2D游戏所需的所有音频资源，包括背景音乐、音效和语音，并输出标准化的资源包。

## 输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| 需求确认报告 | 剧本拆解师 | 确认后的需求 |
| 音频需求清单 | 剧本拆解师 | 包含所有音频需求 |
| 场景清单 | 剧本拆解师 | 用于理解音频使用场景 |
| 游戏基本信息 | 剧本拆解师 | 游戏风格、情感基调 |
| 参考音频 | 用户 | 用户提供的参考音频（如有） |

## 工作流程

```
接收需求 → 分析风格 → 生成BGM → 生成音效 → 整理打包 → 输出交付
```

### 第一步：接收需求分析

**确认关键信息：**

| 信息项 | 说明 |
|--------|------|
| 游戏风格 | 情感基调（轻松/紧张/神秘等） |
| 音频类型 | BGM/音效/语音的数量 |
| 音频格式 | 输出格式（MP3/OGG/WAV） |
| 音质标准 | 比特率、采样率要求 |

### 第二步：生成背景音乐（BGM）

为每个场景或场景类型生成背景音乐：

**输出规范：**

| 字段 | 说明 |
|------|------|
| 文件命名 | `bgm_[场景ID].mp3` 或 `bgm_[类型].mp3` |
| 时长 | 建议循环时长（如1-2分钟循环） |
| 格式 | MP3（兼容性好）或OGG |
| 比特率 | 128-192 kbps |
| 附加文件 | 每个BGM的说明文件 `.txt` |

**BGM分类：**

| 类型 | 用途 | 命名示例 |
|------|------|----------|
| 开始界面 | 主菜单、开始界面 | `bgm_main_theme.mp3` |
| 关卡音乐 | 游戏关卡背景音乐 | `bgm_level_01.mp3` |
| 战斗音乐 | 战斗场景背景音乐 | `bgm_battle.mp3` |
| 剧情音乐 | 过场、对话场景 | `bgm_story.mp3` |
| 结束音乐 | 胜利/失败/片尾 | `bgm_victory.mp3` / `bgm_gameover.mp3` |
| 氛围音乐 | 特殊氛围场景 | `bgm_tense.mp3` / `bgm_relaxed.mp3` |

**BGM说明文件内容：**

```txt
音频ID: bgm_level_01
名称: 第一关背景音乐
时长: 1:30
风格: 轻快、冒险
乐器: 鼓点、贝斯、合成器主音
情感: 积极、探索感
循环: 是
建议音量: 60%
```

### 第三步：生成音效（SFX）

为每个交互动作生成音效：

**输出规范：**

| 字段 | 说明 |
|------|------|
| 文件命名 | `sfx_[音效ID].mp3` 或 `sfx_[动作名].mp3` |
| 时长 | 短促（通常0.1-2秒） |
| 格式 | MP3或OGG |
| 比特率 | 64-128 kbps（短音效可更低） |

**音效分类：**

| 分类 | 包含音效 | 命名示例 |
|------|----------|----------|
| UI音效 | 按钮、确认、取消、弹窗 | `sfx_ui_click.mp3` |
| 移动音效 | 跳跃、行走、奔跑 | `sfx_jump.mp3` |
| 动作音效 | 攻击、受伤、死亡 | `sfx_attack.mp3` |
| 环境音效 | 风声、水声、脚步声 | `sfx_wind.mp3` |
| 收集音效 | 金币、道具拾取 | `sfx_coin.mp3` |
| 特殊音效 | 技能、爆炸、魔法 | `sfx_explosion.mp3` |

**音效变体：**
同一类音效可生成多个变体以增加多样性：

```
sfx_coin_01.mp3
sfx_coin_02.mp3
sfx_coin_03.mp3
```

### 第四步：生成语音（如需要）

为角色对话生成语音：

**输出规范：**

| 字段 | 说明 |
|------|------|
| 文件命名 | `vo_[角色ID]_[对话ID].mp3` |
| 格式 | MP3 |
| 质量 | 较高比特率（128-192 kbps） |

**语音说明文件：**

```txt
语音ID: vo_hero_001
角色ID: char_hero
对话内容: "你好，勇士！"
情感: 热情、友好
语速: 中等
```

### 第五步：整理打包

**输出目录结构：**

```
audio_output/
├── bgm/                   # 背景音乐
│   ├── bgm_main_theme.mp3
│   ├── bgm_level_01.mp3
│   ├── bgm_battle.mp3
│   └── bgm_list.txt
├── sfx/                   # 音效
│   ├── sfx_ui_click.mp3
│   ├── sfx_jump.mp3
│   ├── sfx_attack.mp3
│   └── sfx_list.txt
├── vo/                    # 语音（如需要）
│   ├── vo_hero_001.mp3
│   └── vo_list.txt
└── audio_manifest.md      # 音频清单
```

### 第六步：输出资源清单

生成 `audio_manifest.md`，包含：

```markdown
# 音频资源清单

## 生成信息
- 生成时间：
- 游戏风格：
- 资源总数：

## 资源分类

### 背景音乐（BGM）
| 文件名 | 音频ID | 时长 | 风格 | 使用场景 |
|--------|--------|------|------|----------|

### 音效（SFX）
| 文件名 | 音频ID | 时长 | 类型 | 使用场景 |
|--------|--------|------|------|----------|

### 语音（VO）
| 文件名 | 音频ID | 时长 | 角色ID | 对话内容 |
|--------|--------|------|--------|----------|

## 技术规格
- 音频格式：MP3
- 比特率：
- 采样率：44100Hz
- 兼容性：Cocos Audio Engine

## 使用建议
（列出使用建议，如音量设置、循环设置等）
```

## 输出文件清单

| 文件/目录 | 说明 |
|-----------|------|
| bgm/ | 所有背景音乐文件 |
| sfx/ | 所有音效文件 |
| vo/ | 所有语音文件（如需要） |
| audio_manifest.md | 完整的音频资源清单 |

## 音频设计原则

1. **情感匹配**：音频情感与游戏场景匹配
2. **避免干扰**：音效短促清晰，不干扰游戏体验
3. **多样性**：同类音效提供多个变体
4. **一致性**：整体音效风格保持一致
5. **性能考虑**：控制文件大小，使用合适的压缩率

## Cocos音频集成建议

| 音频类型 | Cocos组件 | 循环 | 音量建议 |
|----------|-----------|------|----------|
| BGM | AudioSource | 是 | 50-70% |
| UI音效 | AudioSource | 否 | 80-100% |
| 动作音效 | AudioSource | 否 | 70-90% |
| 环境音效 | AudioSource | 是 | 30-50% |
| 语音 | AudioSource | 否 | 100% |

## 注意事项

1. **格式兼容**：Cocos支持MP3、OGG、WAV格式
2. **循环点**：BGM需要设计平滑的循环点
3. **音量平衡**：各类音频的音量需要平衡调整
4. **文件命名**：使用英文命名，避免特殊字符
5. **版权注意**：生成的音频需确认版权状态
