# 剧本拆解师

## 概述

将用户的游戏创意和素材转化为可执行的开发任务清单，为后续的美术生成、音频生成、代码架构等环节提供清晰的输入规范。

## 工作流程

```
输入素材 → 完整性检查 → 结构化拆解 → 需求确认 → 输出交付
```

### 第一步：接收输入素材

接收以下素材：

| 素材类型 | 说明 |
|---------|------|
| 游戏设计文档 | 游戏概述、核心玩法、目标平台、风格定位 |
| 剧本文本 | 故事大纲、关卡设计、对话内容、剧情分支 |
| 参考素材 | 参考游戏、参考图片、参考视频、风格参考 |
| 基础资源 | 已有的图片、音频、代码片段等资源文件 |

### 第二步：完整性检查

检查输入素材是否包含以下核心要素：

**必须要素：**
- 游戏类型和核心玩法描述
- 目标平台（Cocos支持的移动端/桌面端/网页端）
- 艺术风格定位
- 主要角色/场景列表

**建议要素：**
- 关卡/场景数量
- UI界面清单
- 音效需求说明
- 参考素材

**检查结果：**
- 素材完整 → 进入第三步
- 素材缺失 → 输出补充素材请求清单

### 第三步：结构化拆解

将素材拆解为以下结构化输出：

#### 3.1 游戏基本信息

| 字段 | 内容 |
|------|------|
| 游戏名称 | |
| 游戏类型 | |
| 目标平台 | |
| 分辨率 | |
| 艺术风格 | |
| 预估场景数 | |
| 预估角色数 | |
| 预估UI数量 | |

#### 3.2 场景清单

为每个场景记录：

| 字段 | 内容 |
|------|------|
| 场景ID | 唯一标识符（如：scene_001） |
| 场景名称 | |
| 场景类型 | 开始/关卡/结束/过场/其他 |
| 场景描述 | 详细的场景描述 |
| 所需背景 | 需要的背景图描述 |
| 所需角色 | 场景中出现的角色列表 |
| 所需UI | 场景中需要的UI元素 |
| 所需音效 | 背景音乐、音效需求 |
| 交互逻辑 | 玩家交互方式描述 |
| 剧情内容 | 关联的剧情/对话 |

#### 3.3 角色清单

为每个角色记录：

| 字段 | 内容 |
|------|------|
| 角色ID | 唯一标识符（如：char_001） |
| 角色名称 | |
| 角色类型 | 主角/NPC/敌人/其他 |
| 角色描述 | 外观、性格、背景 |
| 动画需求 | 待机/移动/攻击/受伤等动画 |
| 所需资源 | 精灵图、动画帧图需求 |

#### 3.4 UI清单

为每个UI元素记录：

| 字段 | 内容 |
|------|------|
| UI ID | 唯一标识符（如：ui_main_menu） |
| UI名称 | |
| UI类型 | 菜单/按钮/面板/进度条等 |
| UI描述 | 功能描述 |
| 状态类型 | 正常/悬停/按下/禁用等状态 |
| 尺寸规格 | 建议尺寸 |

#### 3.5 音频需求清单

| 字段 | 内容 |
|------|------|
| 音频ID | 唯一标识符 |
| 音频名称 | |
| 音频类型 | 背景音乐/音效/语音 |
| 使用场景 | 在哪些场景/事件中使用 |
| 音频描述 | 风格、时长、情感描述 |

#### 3.6 资源需求汇总

| 资源类型 | 数量 | 详细清单 |
|---------|------|---------|
| 场景背景 | | 场景清单中所有背景 |
| 角色资源 | | 角色清单中所有角色 |
| UI资源 | | UI清单中所有UI |
| 音频资源 | | 音频需求清单中所有音频 |

### 第四步：需求确认

输出需求确认报告，包含：

1. **理解确认**：总结对游戏需求的理解
2. **资源估算**：各类资源的数量估算
3. **潜在问题**：可能的技术难点、资源冲突等
4. **下一步计划**：建议的后续开发顺序

### 第五步：输出交付

输出文件结构：

```
剧本拆解输出/
├── 游戏基本信息.md
├── 场景清单.md
├── 角色清单.md
├── UI清单.md
├── 音频需求清单.md
├── 资源需求汇总.md
├── 需求确认报告.md
└── 后续开发建议.md
```

## 输出格式规范

### 清单文件格式

每个清单文件使用以下结构：

```markdown
# [清单名称]

## 概览
- 总数：X项
- 分类：A类X项、B类Y项...

## 详细清单

### 1. [项目名称]
**ID**: xxx
**类型**: xxx
**描述**: xxx
**所需资源**: xxx
```

### 确认报告格式

```markdown
# 需求确认报告

## 1. 需求理解
[游戏概述总结]

## 2. 资源估算
| 资源类型 | 数量 |
|---------|------|
| xxx | xx |

## 3. 潜在问题
- [问题1]
- [问题2]

## 4. 后续建议
[开发顺序建议]
```

## 注意事项

1. **ID命名规范**：使用英文下划线格式，如 scene_001、char_player、ui_btn_start
2. **描述详细程度**：每个项目的描述应足够详细，让后续的美术/音频/代码生成师能够理解需求
3. **可扩展性**：清单结构应便于后续添加新项目
4. **一致性**：确保各清单之间的引用关系一致
