# 场景设计专家

将场景需求转化为详细的美术设计文档。

## 工作流程

```
输入文档                    生成过程                      输出文档
┌─────────────────┐      ┌──────────────┐           ┌─────────────────┐
│ 游戏基本信息.md  │─────▶│  1. 分析场景  │──────────▶│ 大地图设计.htm   │
│ 场景需求.md      │      │  2. 规划区域  │           │ 界面设计.htm     │
│ (可选)设定集.md  │      │  3. 设计图层  │           │ 场景背景设计.htm │
└─────────────────┘      │  4. 配置交互  │           │ 场景物品设计.htm │
                         └──────────────┘           └─────────────────┘
```

## 执行步骤

### 1. 读取输入文档

读取并提取关键信息：
- 游戏类型、视角（俯视/横版/斜45度）
- 场景类型、数量、功能需求
- 界面类型、交互需求
- 技术规格（瓦片尺寸、分辨率）

### 2. 生成大地图设计文档

基于模板 [大地图设计.htm](assets/templates/大地图设计.htm)，填充：

| 章节 | 内容 |
|------|------|
| 地图档案 | ID、名称、类型、章节 |
| 规格要求 | 尺寸、瓦片规格、格式 |
| 地图布局 | 区域划分、关键点位 |
| 瓦片与地形 | 瓦片集规格、地形类型 |
| 图层结构 | 渲染层级规划 |
| 关键点位 | 出生点、传送点、NPC、敌人 |
| 碰撞导航 | 碰撞层、导航网格 |

### 3. 生成界面设计文档

基于模板 [界面设计.htm](assets/templates/界面设计.htm)，填充：

| 章节 | 内容 |
|------|------|
| 界面档案 | ID、名称、类型、触发条件 |
| 规格要求 | 设计尺寸、适配策略 |
| 界面布局 | 整体布局预览、区域划分 |
| UI元素 | 按钮、进度条、技能槽详情 |
| 配色方案 | 主题色、文字色 |
| 交互状态 | 按钮状态、面板动画 |
| 字体规范 | 字体、字号、字重 |

### 4. 生成场景背景设计文档

基于模板 [场景背景设计.htm](assets/templates/场景背景设计.htm)，填充：

| 章节 | 内容 |
|------|------|
| 场景档案 | ID、名称、类型、环境设定 |
| 场景预览 | 完整场景、视平线透视 |
| 图层分解 | 视差层设置、图层规格 |
| 氛围光影 | 时间变化、光源、天气特效 |
| 配色方案 | 主色调、氛围色调 |
| 特殊区域 | 可交互区域标注 |

### 5. 生成场景物品设计文档

基于模板 [场景物品设计.htm](assets/templates/场景物品设计.htm)，填充：

| 章节 | 内容 |
|------|------|
| 物品清单 | 物品总览、尺寸规格 |
| 装饰物品 | 自然装饰、建筑装饰 |
| 可交互物品 | 收集品、功能物品、交互流程 |
| 动画物品 | 动画精灵表、动画配置 |
| 碰撞交互 | 碰撞框设置、交互范围 |

## 设计规范

### ID命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 地图 | map_XXX | map_001 |
| 场景 | scene_XXX | scene_001 |
| UI界面 | ui_XXX | ui_001 |
| 物品 | item_XXX | item_001 |
| 瓦片集 | tileset_XXX | tileset_001 |

### 文件命名规范

```
地图：[ID]_tileset.png / [ID]_map.tmx
界面：[ID]_background.png / [ID]_buttons.png
场景：[ID]_layer_[层级].png
物品：[场景ID]_items_[类型].png
```

### 优先级定义

| 级别 | 含义 | 适用场景 |
|------|------|----------|
| P0 | 必须 | Demo必需、核心功能 |
| P1 | 重要 | 完整游戏必需 |
| P2 | 次要 | 增强体验 |

### 图层规范

| 层级 | 名称 | 内容 | 视差系数 |
|------|------|------|----------|
| 0 | 背景层 | 天空、远景 | 0.1-0.3 |
| 1 | 地面层 | 可行走地面 | 1.0 |
| 2 | 装饰层 | 草丛、石块 | 1.0 |
| 3 | 对象层 | 角色、物品 | 1.0 |
| 4 | 前景层 | 遮挡物 | 0.9 |
| 5 | 效果层 | 光效、天气 | 1.0 |

## HTML输出特点

生成的HTML文档具有以下特点：

1. **清晰的结构分区** - 每个部分都有明确的标题和用途说明
2. **可视化布局** - 用方框标注区域位置和尺寸
3. **交互元素标注** - 按钮状态、动画流程可视化
4. **配色方案展示** - 色块直观展示颜色搭配
5. **表格化规格** - 清晰的技术参数列表
6. **响应式布局** - 适配不同屏幕尺寸
7. **暗色主题** - 便于长时间查看

## 使用示例

**用户输入：**
> 根据场景需求.md 生成大地图设计文档

**执行流程：**
1. 读取场景需求.md，提取场景信息
2. 基于模板生成HTML文档
3. 填充地图布局、区域划分、图层结构
4. 输出到 场景设计输出/ 目录

**输出文件：**
```
场景设计输出/
├── [地图ID]_大地图设计.htm
├── [界面ID]_界面设计.htm
├── [场景ID]_场景背景设计.htm
└── [场景ID]_场景物品设计.htm
```
