# 角色设计专家

将角色需求转化为详细的美术设计文档。

## 工作流程

```
输入文档                    生成过程                      输出文档
┌─────────────────┐      ┌──────────────┐           ┌─────────────────┐
│ 游戏基本信息.md  │─────▶│  1. 分析角色  │──────────▶│ 立绘设计.htm     │
│ 角色需求.md      │      │  2. 确立风格  │           │ 头像表情设计.htm │
│ (可选)设定集.md  │      │  3. 规划配色  │           │ 精灵图设计.htm   │
└─────────────────┘      │  4. 设计构图  │           └─────────────────┘
                         └──────────────┘
```

## 执行步骤

### 1. 读取输入文档

读取并提取关键信息：
- 游戏类型、美术风格
- 角色设定、性格特点
- 势力阵营、世界观背景
- 技术规格要求

### 2. 生成立绘设计文档

基于模板 [立绘设计.htm](assets/templates/立绘设计.htm)，填充：

| 章节 | 内容 |
|------|------|
| 角色档案 | ID、名称、定位、阵营 |
| 规格要求 | 尺寸、分辨率、格式 |
| 外貌特征 | 年龄、体型、发型、瞳色等 |
| 服装设计 | 各部位服装详情 |
| 配色方案 | 主色、辅色、肤色、发色 |
| 立绘构图 | 姿势、重心、图层规划 |
| 表情变体 | 基础表情列表 |
| 输出清单 | 文件命名和规格 |

### 3. 生成头像表情设计文档

基于模板 [头像表情设计.htm](assets/templates/头像表情设计.htm)，填充：

| 章节 | 内容 |
|------|------|
| 规格要求 | 头像尺寸、格式、数量 |
| 基础头像 | 框架、构图要点 |
| 面部特征 | 眼睛、眉毛、鼻子、嘴巴基准 |
| 表情设计 | 基础表情、扩展表情、特殊表情 |
| 变化分解 | 五官变化对照表 |
| 分层建议 | PSD结构规划 |
| 输出清单 | 文件命名和规格 |

### 4. 生成精灵图设计文档

基于模板 [精灵图设计.htm](assets/templates/精灵图设计.htm)，填充：

| 章节 | 内容 |
|------|------|
| 动画清单 | 必需/战斗/交互动画列表 |
| 精灵表布局 | 帧排列可视化 |
| 帧详细说明 | 每帧姿势描述 |
| 锚点与碰撞 | 锚点位置、碰撞框规格 |
| 输出规范 | 文件命名、尺寸要求 |

## 设计规范

### ID命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 玩家 | player_XXX | player_001 |
| NPC | npc_XXX | npc_001 |
| 敌人 | enemy_XXX | enemy_001 |
| BOSS | boss_XXX | boss_001 |

### 文件命名规范

```
立绘：[ID]_立绘_全身.png / [ID]_立绘_半身.png
头像：[ID]_avatar_[表情].png
精灵：[ID]_[动画]_[方向].png
```

### 优先级定义

| 级别 | 含义 | 适用场景 |
|------|------|----------|
| P0 | 必须 | Demo必需、核心功能 |
| P1 | 重要 | 完整游戏必需 |
| P2 | 次要 | 增强体验 |

### 配色建议

| 阵营 | 色调 | 主色示例 |
|------|------|----------|
| 科技派 | 冷色调 | 蓝、灰、银、霓虹 |
| 魔法派 | 暖色调 | 红、金、紫、绿 |
| 中立 | 中性色 | 棕、白、黑 |

## HTML输出特点

生成的HTML文档具有以下特点：

1. **清晰的结构分区** - 每个部分都有明确的标题和用途说明
2. **可视化占位符** - 用方框标注图片位置和尺寸
3. **配色方案展示** - 色块直观展示颜色搭配
4. **表格化规格** - 清晰的技术参数列表
5. **响应式布局** - 适配不同屏幕尺寸
6. **暗色主题** - 便于长时间查看

| [精灵图设计.htm](assets/templates/精灵图设计.htm) | 2D精灵图设计文档 |

## 使用示例

**用户输入：**
> 根据角色需求.md 生成立绘设计文档

**执行流程：**
1. 读取角色需求.md，提取角色信息
2. 基于模板生成HTML文档
3. 填充角色设定、配色方案、构图建议
4. 输出到 角色设计输出/ 目录

**输出文件：**
```
角色设计输出/
├── [角色ID]_立绘设计.htm
├── [角色ID]_头像表情设计.htm
└── [角色ID]_精灵图设计.htm
```
